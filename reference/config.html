    Config Reference¶

    Main¶

    json CsmlConfig : object¶
    Required members:¶
    source : ScivalConfig | WosConfig | PureConfig | ElibraryConfig¶

    Source-specifc configuration

    schema : array of string¶

    List of SQL scripts initializing the CSML schema

    export : array of string¶

    List of SQL scripts copying data from the tmp schema into the main schema

    Scival¶

    json ScivalConfig : object¶
    Required members:¶
    type : "scival"¶
    glob : string¶

    A glob pattern for finding CSVs, such as ./folder/*.csv

    header_length : integer | "auto"¶

    number of lines to skip, or auto for auto-detection

    fields : object¶
    Fields that will be imported into the database for further processing
    The key is the CSV field name, and the value is the column name in sqlite table

    The recommended way to generate those is by checking your CSV
    (with check-scival-fields) against and existing config, then renaming
    whatever fields have changed
    category_mapping : object¶

    Mapping a column name from fields to the corresponding csml_record_category.type_category id

    Optional members:¶
    authors : AuthorsConfig | null = null¶

    Required members:¶
    glob : string¶

    json AddressParams : object¶
    Optional members:¶

    json UssrParams : object¶
    Optional members:¶
    republics : array of string = []¶

    List of Soviet republics

    Since ordering is inconsistent, the parser needs to distinguish them from city names
